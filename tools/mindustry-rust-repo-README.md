# mindustry-rust

这是 `mindustry-rust` 的对外仓库。

它的作用不是单纯存放一份源码镜像，而是持续追踪 Rust 版 Mindustry 的已完成成果、正在推进的实现、测试资源，以及可直接复用的运行/打包/校验脚本。

## 这个仓库适合谁

- 想直接查看 Rust 版当前做到什么程度的人
- 想拉代码、跑测试、打包当前可用版本的人
- 想跟进后续重构、排查问题、追踪提交历史的人

## 当前包含什么

- Rust 客户端相关实现代码
- 与 Java 原版对拍用的测试和夹具
- Windows 侧常用的打包、校验、发布脚本
- 用于同步、交接、审查的辅助文档

## 当前项目状态

这个项目正在持续推进，目标是最终实现一个以 Rust 为内核、并尽量与原版 Mindustry 保持外部表现一致的客户端。

目前更准确的理解方式是：

- 仓库已经具备持续提交、持续校验、持续同步的基础能力
- Rust 侧已经覆盖了一部分高优先级协议、快照、输入和展示链路
- 还不能宣称已经与原版 Java 桌面客户端完全等价

如果你准备直接使用，请先按下面的阅读顺序确认当前能力边界，而不是默认它已经覆盖原版全部功能。

## 建议阅读顺序

1. [`tools/README.md`](tools/README.md)
2. [`tools/WINDOWS-RELEASE.md`](tools/WINDOWS-RELEASE.md)
3. [`tools/MINDUSTRY-RUST-HANDOFF.md`](tools/MINDUSTRY-RUST-HANDOFF.md)
4. `audit/` 下的审查文档

## 目录说明

- `rust/`: Rust 侧主要实现代码
- `tests/`: Java/Rust 对拍测试与资源
- `fixtures/`: 运行、校验和打包会用到的夹具
- `tools/`: 面向使用、打包、同步、校验的脚本和说明
- `audit/`: 审查结论、风险记录、后续跟进文档

## 说明

这个仓库会继续快速变化。README 会尽量优先回答“现在能不能用、该先看哪里、哪些能力已经落地”这些用户问题；更细的发布治理、审查记录和实现边界，请继续查看 `tools/` 和 `audit/`。
